# 2026-04-17 学习笔记

## AutoGLM 手机智能体框架详解

### 问题
AutoGLM 是什么？它的架构和工作原理是什么？如何使用？

### 答案
AutoGLM 是智谱AI开发的开源手机Agent框架，能让AI通过自然语言指令自动操作手机完成各种任务。

**核心工作流程：**
1. 用户输入自然语言指令（如"打开小红书搜索美食攻略"）
2. 系统截取手机屏幕截图
3. 视觉语言模型（VLM）分析屏幕内容并输出下一步操作
4. ADB/HDC/iOS工具执行操作（点击、滑动、输入等）
5. 循环执行直到任务完成

**项目结构：**
```
phone_agent/
├── agent.py          # PhoneAgent主类，编排整个Agent循环
├── agent_ios.py      # iOS版本的Agent
├── device_factory.py # 设备工厂，支持ADB/HDC/iOS
├── actions/          # 操作处理模块
│   └── handler.py    # 解析和执行各种操作（Tap、Type、Swipe等）
├── adb/              # Android设备控制
│   ├── connection.py # ADB连接管理（本地/远程）
│   ├── screenshot.py # 屏幕截图
│   ├── input.py      # 文本输入（ADB Keyboard）
│   └── device.py     # 设备控制（点击、滑动等）
├── hdc/              # 鸿蒙设备控制（类似adb/）
├── xctest/           # iOS设备控制（WebDriverAgent）
├── config/           # 配置文件
│   ├── apps.py       # 50+中国主流应用的包名映射
│   ├── prompts_zh.py # 中文系统提示词
│   ├── prompts_en.py # 英文系统提示词
│   └── timing.py     # 操作延时配置
└── model/            # AI模型客户端
    └── client.py     # OpenAI兼容API调用
```

### 关键点

#### 1. 模型服务选项
- **智谱BigModel API**（推荐）：`https://open.bigmodel.cn/api/paas/v4`，模型名 `autoglm-phone`
- **ModelScope API**：`https://api-inference.modelscope.cn/v1`，模型名 `ZhipuAI/AutoGLM-Phone-9B`
- **本地部署**：使用vLLM或SGLang，需要24GB+显存的GPU

#### 2. 设备支持
- **Android**：通过ADB控制，需要安装ADB Keyboard用于文本输入
- **鸿蒙（HarmonyOS）**：通过HDC控制，使用原生输入法
- **iOS**：通过WebDriverAgent（XCTest）控制

#### 3. 核心操作指令
| 操作 | 格式 | 说明 |
|------|------|------|
| Launch | `do(action="Launch", app="xxx")` | 启动应用 |
| Tap | `do(action="Tap", element=[x,y])` | 点击坐标（0-999归一化） |
| Type | `do(action="Type", text="xxx")` | 输入文本 |
| Swipe | `do(action="Swipe", start=[x1,y1], end=[x2,y2])` | 滑动操作 |
| Back | `do(action="Back")` | 返回上一页 |
| Home | `do(action="Home")` | 回到桌面 |
| Long Press | `do(action="Long Press", element=[x,y])` | 长按 |
| Double Tap | `do(action="Double Tap", element=[x,y])` | 双击 |
| Wait | `do(action="Wait", duration="x seconds")` | 等待加载 |
| Take_over | `do(action="Take_over", message="xxx")` | 请求人工接管（登录/验证码） |
| finish | `finish(message="xxx")` | 结束任务 |

#### 4. 系统提示词设计（关键！）
- 模型输出格式：`<think>思考过程</think><answer>操作指令</answer>`
- 坐标系统：左上角(0,0)到右下角(999,999)，归一化处理
- 18条操作规则：包括检查当前app、处理加载失败、滑动查找、敏感操作确认等
- 支持中英文两套提示词

#### 5. 命令行使用方式
```bash
# 交互模式（Android）
python main.py --base-url https://open.bigmodel.cn/api/paas/v4 --model "autoglm-phone" --apikey "your-key"

# 指定任务
python main.py --base-url http://localhost:8000/v1 "打开美团搜索附近的火锅店"

# 鸿蒙设备
python main.py --device-type hdc --base-url http://localhost:8000/v1

# iOS设备
python main.py --device-type ios --wda-url http://192.168.1.100:8100

# 列出支持的应用
python main.py --list-apps

# 检查模型部署
python scripts/check_deployment_cn.py --base-url http://localhost:8000/v1 --model autoglm-phone-9b
```

#### 6. Python API使用
```python
from phone_agent import PhoneAgent
from phone_agent.model import ModelConfig
from phone_agent.agent import AgentConfig

# 配置模型
model_config = ModelConfig(
    base_url="http://localhost:8000/v1",
    model_name="autoglm-phone-9b",
    temperature=0.1,
)

# 配置Agent
agent_config = AgentConfig(
    max_steps=50,
    verbose=True,
    lang="cn",
)

# 创建Agent
agent = PhoneAgent(
    model_config=model_config,
    agent_config=agent_config,
)

# 执行任务
result = agent.run("打开淘宝搜索无线耳机")
print(result)
```

#### 9. 高级功能
- **自定义回调**：敏感操作确认、人工接管处理
- **单步调试**：`agent.step()` 方法用于调试
- **批量任务**：循环执行多个任务
- **iOS支持**：通过WebDriverAgent，需要配置Xcode工程

#### 10. 常见问题解决
- **设备未找到**：重启ADB服务、检查USB调试、更换数据线
- **能打开应用但无法点击**：开启「USB调试(安全设置)」
- **文本输入不工作**：确保ADB Keyboard已安装并启用
- **截图黑屏**：敏感页面（支付/银行类应用），系统会自动处理
- **Windows编码异常**：设置 `PYTHONIOENCODING=utf-8`
