### 1.4 核心能力模块速览

| 能力 | 是什么 | 触发方式 | 示例 |
|------|--------|---------|------|
| Tools（工具调用） | Claude 读写文件、执行命令的能力 | 模型自动决策 | Read/Write/Bash |
| Hooks（生命周期钩子） | 在工具调用前后插入自定义逻辑 | 配置文件声明 | PreToolUse / PostToolUse |
| Skills（自定义技能） | 预设的工作流指令集 | `/skill-name` | `/git-commit` |
| Agents（子代理） | 独立执行子任务的 Claude 实例 | Agent tool | 并行代码审查 |
| Slash Commands（自定义命令） | 用户自定义的快捷指令 | `~/.claude/commands/` | `/my-first-command` |
| CLAUDE.md（项目记忆） | 跨会话持久化的项目上下文 | 自动加载 | 代码风格约束 |
| MCP（外部工具集成） | 接入外部服务的协议 | `.mcp.json` 配置 | 数据库 / 浏览器 |

### 1.5 实战示例合集

## Part 2：提示词输入后的完整流程图解

### 2.1 七层流转架构图

```
用户输入提示词
  ↓ [1] CLI 解析层        → 解析命令、加载 CLAUDE.md、读取上下文
  ↓ [2] 上下文构建层      → 合并项目记忆、会话历史、系统 prompt
  ↓ [3] API 请求层        → 打包发送到 Anthropic API
  ↓ [4] 模型推理层        → Claude 决策：直接回答 or 调用工具
  ↓ [5] 工具调用层        → 执行 Read/Write/Bash/Agent...
  ↓ [6] Hook 拦截层       → PreToolUse/PostToolUse 钩子触发
  ↓ [7] 输出返回层        → 流式返回结果，写入文件或打印
```

### 2.2 每层详解
每一层包含：
- **做了什么**（1-2 句）
- **为什么这样设计**（原理，不少于 2 句）
- **开发者可以在这一层做什么**（可扩展点）
- **示例**（代码或配置片段）

### 2.3 端到端追踪示例
以「帮我重构这个函数」为例，从用户输入到文件写入，逐步追踪每一层发生的事情，标注：
- 🤖 Claude 自动决策的节点
- 🔧 开发者可干预的节点

💡 **初学者最常见误区**：以为模型「直接修改文件」，实际上模型只负责决策，文件读写是通过 Tools 层执行的，Hook 层可以在执行前拦截审查。
