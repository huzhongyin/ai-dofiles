# identify-intent

在会话结束或想要回顾本次交互时运行，自动识别意图类别并将知识归档到对应位置。

---

## 本次会话描述（可选）

$ARGUMENTS

---

## 执行步骤

### Step 1：回顾本次会话

回顾当前会话的主要交互内容，识别出主导意图类别：

| 意图类型 | 占比 | 关键交互 |
|---------|------|---------|
| （从对话历史中提取） | ... | ... |

输出格式：
```
【会话意图分析】
主导意图：BUG_FIX / FEATURE_IMPL / UX_OPTIMIZE / LEARNING / ARCHITECTURE / REVIEW / RETROSPECTIVE
次要意图（如有）：...
关键交互摘要：<2-3句话>
```

---

### Step 2：按意图类型路由归档

根据识别到的意图，决定归档内容和位置：

| 意图 | 归档内容 | 目标文件 |
|------|---------|---------|
| `BUG_FIX` | Bug描述+根因+修复模式 | 项目 `TESTING.md` Bug登记册 |
| `FEATURE_IMPL` | 新增结构/文件/依赖 | 项目 `README.md` |
| `UX_OPTIMIZE` | 新增UI约束 | 项目 `Agent.md` `<约束条件>` |
| `LEARNING` | 学习内容文档 | `~/.claude/learning/YYYY-MM-DD-<topic>.md` |
| `ARCHITECTURE` | 决策摘要+选择理由 | `~/.claude/decisions/YYYY-MM-DD-<topic>.md` |
| `REVIEW` | 发现的问题+改进建议 | `~/.claude/reviews/YYYY-MM-DD-<topic>.md` |
| `RETROSPECTIVE` | 按类型分发到以上各目标 | 多目标路由 |

**无复用价值的一次性交互 → 不归档。**

#### 补充规则：Claude 生成内容的归档判断

除了用户的学习行为，**Claude 在会话中生成的输出**也需要独立评估复用价值。按以下标准判断：

| 输出类型 | 归档条件 | 归档目标 |
|---------|---------|---------|
| 技术教程 / 学习文档 | 有完整结构、针对用户定制、可直接复用或分享 | `~/.claude/learning/` |
| 架构设计 / 方案对比 | 有决策依据、影响后续开发 | `~/.claude/decisions/` |
| 代码片段 / 配置示例 | 可直接复用，不依赖当前会话上下文 | `~/.claude/learning/` 对应主题文件中追加 |
| 流程图 / 原理解析 | 解释了非显而易见的机制，有长期参考价值 | `~/.claude/learning/` |
| 一次性问答 | 无结构，不可复用 | ❌ 不归档 |

**判断口诀**：「如果这个输出从对话记录里消失了，我还需要重新生成它吗？需要 → 归档；不需要 → 跳过。」

---

### Step 3：用户画像更新评估

评估本次会话是否反映了用户能力/偏好的变化：

- 是否发现了用户在某领域**明显提升**或**新技能**？
- 是否发现了新的**工作偏好**或**习惯模式**？
- 是否需要更新 `~/.claude/profile/USER_PROFILE.md`？

输出格式：
```
【画像更新建议】
是否需要更新：是 / 否
如需更新，变化点：
- USER_PROFILE.md：<具体字段和新内容>
- PREFERENCES.md：<如有变化>
```

---

### Step 4：输出归档草稿

为每个需要更新的文件，输出完整的「待添加内容」：

```
━━━ 文件：<相对路径或绝对路径> ━━━
<插入位置说明>

<直接可粘贴的新增内容>

━━━ 结束 ━━━
```

---

### Step 5：直接执行写入

**不需要等待用户确认**，直接使用 Edit/Write/Bash 工具将草稿内容写入对应文件。

写入完成后，输出一条简要的执行摘要：
```
【归档完成】
✅ <文件路径> — 已追加 <内容类型>
✅ <文件路径> — 已更新 <字段名>
（跳过）<文件路径> — 无复用价值
```
