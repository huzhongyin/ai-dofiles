- 优先级: 中
- 最后更新: 2026-04-08

## 内容
车端文件传输（相机照片等）使用独立的 **TCP Socket 直连**，不走 BLE 或 MQTT 通道：

- **实现**: 纯 OC（`XPVehicleFileTransferManager` + `XPCarFileTransferTcpSocket`）

**核心规则**: 不要尝试将文件传输改为 BLE 或 MQTT 通道，带宽和延迟不满足大文件需求。

**代码位置**: `submodule/XPVehicleExt/XPVehicleExt/FileTransfer/`

## 内容
Polling 无感解锁/闭锁需要 App 在后台持续运行。保活机制依赖 iBeacon：

# Polling 双通道（BLE + MQTT）车态数据依赖

## 元信息
- 触发关键词: ["drive.shift_state", "vehicleStateChanged", "gearValueIsSupportPolling", "双通道", "BLE档位", "MQTT档位", "Polling暂停"]
- 分类: 业务设计
- 优先级: 高
- 最后更新: 2026-04-21

## 内容

Polling 的 `gearValueIsSupportPolling` 状态同时依赖 BLE 和 MQTT 两个通道推送的 `drive.shift_state`：

1. **BLE 通道**: 车端 CAN 数据 → XPBLEDataProcessor DataParser → `buildDriveMessage` → `drive.shift_state`
2. **MQTT 通道**: 车端 → 云端 → MQTT → TCP 推送 → `drive.shift_state`

两个通道任一推送 `drive.shift_state` 变化，都会触发 `PLXPMultiBleManager.vehicleStateChanged:` 更新 `gearValueIsSupportPolling`。

### 风险点
- 若 BLE DataParser 存在路由 Bug（如 D55DataParser 的 `D55CanType_App_Msg1` 错误路由），BLE 通道将无法推送档位
- 此时若 MQTT 也断连，Polling 状态将无法恢复，导致感应闭锁/解锁失效

- 最后更新: 2026-04-08

## 内容
Polling 系统定义了 9 级同心圆距离模型（`PLBLEKeyDistanceLevel`），从近到远：

```
车辆中心
  ← InCar (车内)
  ← 1m圈 (OneMetreCircle)
  ← 解锁圈 (UnlockCircle)     ← 靠近到此距离触发解锁
  ← 授权圈 (EnableCircle)
  ← 缓冲圈 (BufferCircle)     ← 防止乒乓抖动的缓冲区
  ← 闭锁圈 (LockCircle)       ← 远离到此距离触发闭锁
  ← 迎宾圈 (WelcomeCircle)

# Polling 与普通车控指令码差异

## 元信息
- 触发关键词: ["PLXPTBoxCommand", "PollingUnlock", "Polling指令", "TBox指令", "解锁指令码", "Polling解锁"]
- 分类: 业务逻辑
- 优先级: 高
- 最后更新: 2026-04-08

## 内容
Polling 解锁和普通车控解锁走的是**不同的指令码**：

| 操作 | Polling 指令 | 普通车控指令 |
|------|-------------|-------------|
| 解锁 | `PLXPTBoxCommandPollingUnlock` | `PLXPTBoxCommandUnlock` / `BLEBoxCommandUnlock(0x01)` |
| 闭锁 | `PLXPTBoxCommandPollingLock` | `PLXPTBoxCommandLock` / `BLEBoxCommandLock(0x02)` |

**PLXPTBoxCommand** 共约 30 个指令，包括：解锁/闭锁/后备箱/充电口/滑门/剪刀门/泊车等。

**核心规则**: 混用指令码会导致车端识别错误的指令来源（无感 vs 手动），可能影响计费、安全策略等。

# 双通道故障放大效应

## 元信息
- 触发关键词: [DataParser, BLE, MQTT, 双通道, 故障放大, shift_state, gearValueIsSupportPolling]
- 分类: 领域知识
- 优先级: 高
- 最后更新: 2026-04-22

## 内容

### 现象
单通道数据缺失时，另一通道断连会导致功能完全失效。

Polling 状态（如 `gearValueIsSupportPolling`）同时依赖 BLE 和 MQTT 两个通道推送的 `drive.shift_state`。当 BLE 通道因 DataParser Bug 无法推送档位，而 MQTT 断连后也停止推送，Polling 状态将无法恢复。

排查此类问题时必须同时分析 BLE 和 MQTT 两个数据通道，不能仅分析其中一个。

# BLE data 与 tboxData 通道分发

- 触发关键词: XPBLEData, data, tboxData, BLE通道, TBox通道
- 分类: 蓝牙通信
- 优先级: high
- 最后更新: 2026-04-22

## 要点

`XPBLEData` 有两个数据字段，对应不同通道：
- `.data` → BLE 通道发送
- `.tboxData` → TBox/MQTT 通道发送

## 典型问题

将 `tboxData` 通过 BLE 发送会导致帧格式不匹配，车端收到指令但无法执行或执行错误操作。

## 规则

严格按通道分发：`.data` 只走 BLE，`.tboxData` 只走 TBox。不可混用。

# BaseVehicle 协议组合模式

## 元信息
- 触发关键词: ["BaseVehicle", "VehiclePresenter", "VehicleCommunication", "VehiclePresenterExt", "协议组合", "车辆协议"]
- 分类: 领域知识
- 优先级: 中
- 最后更新: 2026-04-08

## 内容
`BaseVehicle` 通过协议组合模式定义"一辆车"的完整能力：

```
BaseVehicle = VehiclePresenter + VehiclePresenterExt + VehicleCommunication
```

| 子协议 | 职责 | 关键 API |
|--------|------|----------|
| VehiclePresenter | 展示层能力 | vehicleState / sendControlCommand / connect / disconnect / events |
| VehiclePresenterExt | 扩展能力 | RSSI 观察 / Polling 代理 |
| VehicleCommunication | 通信能力 | vin / uid / tcpConnection / bleConnection / cacheManager |

# 存量拼写错误不要修改

## 元信息
- 触发关键词: ["VEVeihcle", "拼写错误", "存量代码", "remuseBlePolling", "重命名风险"]
- 分类: 工程规范
- 优先级: 中
- 最后更新: 2026-04-08

## 内容
项目存量代码中存在已知拼写错误，**不建议修改以免影响引用**：

| 错误 | 正确 | 位置 |
|------|------|------|
| `VEVeihcleSDKTool` | VEVehicleSDKTool | XPVehicleExt/Utilities/ |
| `remuseBlePolling` | resumeBlePolling | XPPollingSDK 生命周期协议 |

**原因**: 这些名称已在多处被引用（其他 submodule、主工程、OC/Swift 互操作），修改会触发大范围级联变更。

**正确做法**: 新增代码使用正确拼写，存量代码保持不变。如需重命名，需评估影响范围后专门立项。

# GRDB 数据库使用规范

## 元信息
- 触发关键词: ["GRDB", "XPDBManager", "数据库", "BleKeyCard表", "数据库迁移", "XPDBClient"]
- 分类: 工程规范
- 优先级: 中
- 最后更新: 2026-04-08

## 内容
项目使用 GRDB 进行本地数据持久化，统一由 `XPDBManager` 管理：

**注册的表**:
1. `BleKeyCard` — 蓝牙钥匙卡（16 种状态、16 种类型）
2. `CertKeyCard` — 证书钥匙卡
3. `CertKeyMessage` — 证书钥匙消息
4. `ChargeSearchHistoryModel` — 充电搜索历史

**使用规范**:
- 数据模型需实现 GRDB 的 `FetchableRecord` + `PersistableRecord` 协议
- 表结构变更走数据库迁移（`BleKeyCard` 已有迁移支持）
